5-Comentem sobre a importância de usar code review! (No documento).

O code review faz parte do processo de desenvolvimento, é uma rotina que deve ser adotada a fim de colaborar na entrega de um bom código, com qualidade, para seguir para as próximas etapas de desenvolvimento do software. 
O code review é uma boa prática, onde se faz revisões do código de dada aplicação, em geral, o code review é realizado a cada commit, pois estes dizem respeito às alterações que o código sofre ao longo da atividade de 
desenvolvimento, seja por ajustes ou implementação de uma nova funcionalidade da aplicação.
É importante manter a prática de realizar code review, pois nesta fase, em geral, possíveis bugs, falhas ou pontos de melhorias são levantados e ajustados antes mesmo do código chegar em produção. 
É uma atividade rápida de se realizar e que enriquece a qualidade do código sem afetar o tempo de entrega.
Durante a atividade usar checklist, automatizar os testes, ficar atento a possíveis impactos e novas ameaças, são práticas que podem ser adotadas para auxiliar sua execução.

Abaixo listamos algumas razões pelas quais o code review é importante para a plataforma YouTube:

1.Melhoria na qualidade do código: com um segundo par de olhos analisando o código, é possível encontrar e corrigir problemas que poderiam passar despercebidos pelo desenvolvedor que escreveu o código originalmente. 
Isso pode levar a uma redução no número de bugs e a um código mais limpo e organizado.
2.Aprendizado contínuo: o code review pode ser uma oportunidade para que desenvolvedores aprendam com seus colegas e adquiram novas habilidades ou conhecimentos.
3.Consistência no código: o code review também ajuda a garantir que o código esteja em conformidade com os padrões e as melhores práticas da equipe de desenvolvimento. 
Isso pode levar a um código mais consistente e mais fácil de manter.
4.Prevenção de conflitos: o code review pode ajudar a prevenir conflitos entre diferentes partes do código, especialmente quando vários desenvolvedores trabalham no mesmo código ou no mesmo recurso.
5.Melhoria na comunicação e colaboração: o processo de code review pode promover a comunicação e a colaboração entre os desenvolvedores, especialmente quando os comentários e as sugestões de melhorias são fornecidos de forma construtiva.
Em resumo, o code review é uma prática importante para a plataforma YouTube, pois pode levar a um código de maior qualidade, a um aprendizado contínuo, à consistência do código, à prevenção de conflitos e à melhoria na comunicação e 
colaboração entre os desenvolvedores. É recomendado que o processo de code review seja incorporado à cultura da equipe de desenvolvimento como uma prática rotineira e constante.
