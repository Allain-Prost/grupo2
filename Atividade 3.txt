Descrição do aplicativo:
O YouTube é uma plataforma de compartilhamento de vídeos online que permite aos usuários enviar, visualizar, avaliar, comentar e compartilhar vídeos. O YouTube é uma empresa da Google LLC, que foi fundada em 2005.

Padrões de código e nomenclatura:
1 - Desenvolvimento:
Para o desenvolvimento do aplicativo do YouTube, recomenda-se como definir um conjunto de diretrizes que orientam o desenvolvimento de código consistente e legível em todo o projeto. 
O objetivo é garantir que o código seja fácil de entender, manter e escalar, independentemente de quem o desenvolva ou mantenha. Para tanto, faz-se necessário estabelecer padrões claros de nomenclatura de classes, 
métodos, variáveis e funções, que sejam descritivos e concisos.
EX: poderíamos utilizar nomes autoexplicativos como "UserController" para uma classe que gerencia usuários, "getVideoDetails" para um método que retorna detalhes de um vídeo 
ou "thumbnailImage" para uma variável que armazena a imagem em miniatura de um vídeo.

2 - Documentação:
Como documentação, ao longo de toda atividade de desenvolvimento se faz necessário usar comentários para explicar o propósito do código e como ele funciona.

3 - Padrão de formatação de código:
Podemos utilizar indentação, espaçamento e alinhamento, para que o código seja legível e organizado. Afim de ganhar tempo durante o desenvolvimento, podemos fazer uso de ferramentas de automatizadas de formatação, como por exemplo 
Prettier ou o ESLint, para garantir que o código siga as diretrizes de formatação.Além disso, podemos usar o idioma inglês como padrão do código, uma vez que as próprias funções, métodos e bibliotecas do python são em inglês.

4 - Padrão para estrutura de pastas e arquivo:
A fim de manter a organização do projeto, ficou decidido manter nomes de pastas para cada tipo de arquivo.
A pasta "models" para arquivos relacionados a modelos de dados, uma pasta "controllers" para arquivos relacionados a lógica de controle, uma pasta "views" para arquivos de visualização.

5 - Escrever funções simples e modularizadas, que executem uma única tarefa.

6 - Sempre que possível, usar bibliotecas de terceiros quando possível, em vez de escrever código personalizado, para economizar tempo e evitar erros comuns.

7 - Fazer testes de unidade e integração para garantir que o código esteja funcionando corretamente.Se possível utilizando o pytest como ferramenta de automação.

8 - Usar controle de versão para rastrear mudanças no código e colaborar com outros desenvolvedores.
